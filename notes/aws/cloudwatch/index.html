Basics

  • Globally resilient, secure across all AWS regions
  • Own dedicated instance of IAM in each account
  • Full trust between account and IAM
  • Users
    • Humans or applications that need access to AWS account
  • Groups
    • Collections of related users
  • Roles
    • Can be used by AWS services or for granting external access to the account
    • Used for uncertain number of entities that may need access
  • Policies
    • Allow or deny access to AWS services only when attached to a user, group, or role
  • IAM is an identity provider (IDP)
  • Authenticates identities
  • Then authorizes
  • No costs but there are limits
  • No direct control on external accounts or users
  • Identity federation and MFA
